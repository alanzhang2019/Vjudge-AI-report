**为了完成这一部分，我假设你正在尝试下面这道全国联赛常见题进行示范：**

**题目（虚拟示例）：** —— **有一棵树，求所有点对 (u, v) 的距离之和。（N <= 2000 分）**

**a) AI 题解摘要**：本题的核心是计算每条边被经过的次数。一条边 (u, v) 会把树分成两部分，大小为 `size_u` 和 `n - size_u`，那么这条边的贡献就是 `边权 * size_u * (n - size_u)`。

**b) 暴力思路：**
- **怎么做？** 对树上每个点进行DFS，计算它到所有其他点的距离。所有距离累加。
- **复杂度**：O(N^2)。因为每个点需要O(N)时间。
- **能拿部分分？** 可以拿30-50分（取决于数据范围）。N=2000可以全拿满分。但对N=1e5，这是完全不可行的。

**c) 瓶颈在哪里？**
- **时间瓶颈**：重复计算。对每个点都进行遍历，把整棵树遍历了N次，做了大量无意义的重复工作。
- **空间瓶颈**：N=2000时没有空间问题。

**d) 关键性质/不变量观察**
- **关键观察**：这个问题的答案等价于 **每条边贡献的次数**。
- 想象一下，一条边把树分成左右两部分。任何一条从一个部分到另一个部分的路径，都必须经过这条边。
- 所以，这条边被经过的次数 = 左边部分点的数量 * 右边部分点的数量。
- 我们需要的就是找到一种方法，把N次遍历分解成一次DFS。

**e) 最终正解的推导：**
1. **一次DFS求子树大小**：从任意根（比如1）开始做一次DFS。在遍历过程中，计算出每个节点子树的大小 `sz[v]`。
2. **计算每条边的贡献**：当我们递归到节点 `v` 时，它到父节点 `u` 的边权为 `w`。那么这条边把树分成了大小为 `sz[v]` 和 `n - sz[v]` 的两部分。
3. **累加贡献**：答案 += `w * sz[v] * (n - sz[v])`。
4. **核心代码结构（伪代码）：**
```cpp
#include <bits/stdc++.h>
using namespace std;
const int N = 1e5 + 5;
vector<pair<int, int>> g[N]; // 邻接表存图 (节点, 边权)
int n, sz[N];
long long ans = 0;

void dfs(int u, int fa) {
    sz[u] = 1; // 包含自己
    for (auto &[v, w] : g[u]) {
        if (v == fa) continue;
        dfs(v, u);
        // 边权为 w，子树大小为 sz[v]
        ans += 1LL * w * sz[v] * (n - sz[v]);
        sz[u] += sz[v];
    }
}

int main() {
    ios::sync_with_stdio(false); cin.tie(0); cout.tie(0);
    cin >> n;
    for (int i = 1, u, v, w; i < n; ++i) {
        cin >> u >> v >> w;
        g[u].emplace_back(v, w);
        g[v].emplace_back(u, w);
    }
    dfs(1, 0);
    cout << ans << '\n';
    return 0;
}
```
**对比你的代码习惯：** 注意我使用了 `auto &[v, w]`（这是C++17的structured binding），取代你以前 `pii p=g[u][i]; int v=p.first;`，代码精简了很多。同时变量命名清晰（`u`, `v`, `w`, `sz`, `ans`），完全可读。
