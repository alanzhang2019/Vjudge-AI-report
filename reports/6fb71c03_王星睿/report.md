    - **关键**：手工计算乘除法的核心思想：从低位向高位，逐位运算，并处理进位/借位。
    - **不变量**：过程中的进位/借位值。对于除法，每一次试商得到的余数，是下一轮的被除数的一部分。

    - **核心代码**（高精度加法示例）：
      ```cpp
      vector<int> add(vector<int> a, vector<int> b) {
          // 确保 a 的位数 >= b 的位数
          if (a.size() < b.size()) return add(b, a);
          vector<int> res;
          int carry = 0;
          for (int i = 0; i < a.size(); i++) {
              int sum = a[i] + carry;
              if (i < b.size()) sum += b[i];
              res.push_back(sum % 10);
              carry = sum / 10;
          }
          if (carry) res.push_back(carry);
          return res;
      }
      ```
    - **高精度除法**核心：从高位开始，每次取一部分被除数，除以除数，得到一位商，余数保留给下一位。

##### **题目 3：B4071 [GESP202412 五级] 武器强化**

- **a) AI题解摘要**：这是一道**贪心 + 模拟**题。重点是理解“强化失败不掉级”这个条件，并利用优先队列（最大堆或最小堆）来管理最小值。

- **b) 暴力思路怎么想？**
    - **思路**：枚举所有可能的强化顺序（全排列），计算总消耗，取最小值。
    - **复杂度**：\(O(N!)\)。当 \(N=5\) 时还能算，\(N=100\) 时就完全不可行。
    - **部分分**：能拿到 “小数据” 的部分分。

- **c) 瓶颈在哪里？**
    - **时间**：枚举所有顺序在 \(N\) 稍大时指数爆炸。

- **d) 关键性质/不变量观察 (Key Observation)**
    - **关键**：题目要求“每次选取‘当前’强化等级最高的两个武器”，这不是一个固定顺序！强化会改变武器的等级。这是一个动态过程。
    - **不变量**：强化材料的成本是固定的，你每次用掉的两个最低价值的武器，会合成一个更高级的武器。这本质上类似于**哈夫曼编码**的贪心思想：每次取最小的两个进行合并。

- **e) 最终正解的推导与核心代码结构**
    - **正解**：用 `priority_queue<int, vector<int>, greater<int>>`（最小堆）来维护所有武器的“价值”。每次取出堆顶的两个最小元素（价值最低的），消耗材料，合并成一个新的武器（价值为两者的和 + 消耗的材料），再放回堆中。当堆中只剩一个武器时，结束。这次模拟的总消耗就是答案。
    - **核心代码**：
      ```cpp
      #include <bits/stdc++.h>
      using namespace std;
      int main() {
          int n, m;
          cin >> n >> m;
          priority_queue<int, vector<int>, greater<int>> pq; // 最小堆
          for (int i = 1; i <= m; i++) {
              int p, c;
              cin >> p >> c;
              pq.push(c); // 价值入堆
          }
          
          long long total_cost = 0;
          // 当堆中元素数量 > 1 时，始终进行合并
          while (pq.size() > 1) {
              int a = pq.top(); pq.pop();
              int b = pq.top(); pq.pop();
              int new_value = a + b;  // 合并后的新武器的基础价值？
              // 注意：根据题意，消耗的材料就是 c，我们不需要把 c 加到总消耗里吗？
              // 仔细看题：每次消耗一把对应材料，这把材料的价值是c。
              // 所以每次我们消耗的不是a和b的价值，而是c。
              // 因此，总消耗 = m种材料的价值之和。因为我们进行了m次强化？
              // 不是，我们每次只消耗一种材料，但材料种类有m种。
              // 我们需要选一种方式来使用材料，使得总消耗最小。
              // 看懂题意：有m种材料，每种有p个，单个价值c。
              // 每次从所有武器中选两个等级最高的，然后选择一个材料进行强化，花费材料的价值。
              // 所以问题是：我们怎么选择材料，使得总花费最少？
              // 贪心：每次选两个等级最高的武器，然后用最便宜的可选材料强化。
              // 但我们有优先队列维护武器等级？错！武器按等级排，但我们选“等级最高的”。
              // 那我们就需要用最大堆来维护“当前等级最高的两个武器”。
              // 每次取出堆顶的两个武器（等级最高），消耗当前可用材料中最便宜的一个，强化它们。
              // 强化后，它们的等级+1，总价值 = 原价值之和 + 材料价值？
              // 题面说：“强化后，被强化的两把武器将融为一体，变为等级+1的新武器，且其等级与两把武器之和相等。”  这里的“有一定概率”？题面未提及概率，应是确定的。
              // 其价值 = 原两武器价值之和 + 材料价值c。
              // 然后新武器入堆。
              // 最终总花费 = 所有使用过的材料价值之和。
          }
      }
      ```
      感觉用最大堆更符合直觉？继续思考。
      
      **关键**: 强化逻辑是：每次从**所有当前存在的武器**中，挑选**等级最高**的两个，然后选择**一种材料**进行强化，材料会消耗掉。强化的总花费=所用材料的价值总和。
      
      所以武器等级是递减的？初始都是等级1。每次强化后，武器等级+1，所以新武器等级最高。因此堆中按等级排序，等级最高的在最上面。这是一个**最大堆**!
      
      **最终结论**: 
      - 用 `priority_queue` 存武器，按**等级从大到小**排序。
      - 用 `multiset` 或另一个 `priority_queue` 存材料，按**价值从小到大**排序。
      - 每次从武器堆顶取两个等级最高武器，从材料堆顶取一个最便宜的材料，进行强化。新武器等级=旧武器等级+1，价值 = 旧武器价值之和+材料价值，入武器堆。
      - 材料消耗掉，从材料堆中弹出。
      - 重复直至只剩1个武器。
      - 总花费 = 所有被使用的材料价值之和。

    - **逻辑**：不是性能瓶颈，而是**理解题意**的瓶颈。尤其是对“统计出现次数、按频率排序、选取前16个作为颜色表、灰度压缩”这一系列流程的执行。

- **d) 关键性质/不变量观察 (Key Observation)**
    - **关键**：题目描述就是最终答案。这是一个“需求文档”，你要做的就是成为一台完美的翻译机，把它翻译成代码。
    - **不变量**：`颜色表`是固定的16色，每种颜色用一个在 `[0, 15]` 的索引代表。

- **e) 最终正解的推导与核心代码结构**
    - **正解**：
        1.  读取图像数据，建立颜色频率统计。频率高的优先。
        2.  按频率从高到低排序，如果频率相同按颜色值从小到大排序。
        3.  选取频率最高的前16种颜色，作为颜色表。如果不够16种，就用颜色值小的补（这题可能不需要，但备着）。
        4.  遍历原始图像，对每个像素点的颜色值，在颜色表中找到最接近的颜色（即差值绝对值最小），并用颜色表中该颜色的索引代替。
        5.  用四位二进制（十六进制）存储索引，并输出压缩后的图像。
    - **核心代码**（步骤2排序）：
      ```cpp
      // 假设我们有一个结构体 Color
      struct Color {
          int value;  // 0-255
          int freq;   // 出现次数
      };
      vector<Color> colors;
      // ... 初始化 colors
      sort(colors.begin(), colors.end(), [](Color a, Color b) {
          if (a.freq != b.freq) return a.freq > b.freq; // 按频率降序
          return a.value < b.value;                     // 频率相同时按颜色值升序
      });
      ```
