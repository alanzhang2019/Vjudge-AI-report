## 6. 【代码质量与工程习惯深度分析】

<p class="text-blue-700 font-semibold">（教练注：因本次导出未包含具体代码，以下分析基于该阶段选手常见的高频错误。请你对照自查，有则改之，无则加勉。）</p>

🔴 **三个必须立刻纠正的坏习惯**

1.  **无优化输入输出与宏定义混乱**
    - **症状**：混合使用`cin/cout`和`scanf/printf`，且未屏蔽同步流，导致大数据输入时神秘TLE。滥用`#define int long long`，导致内存占用翻倍，但又在不需要的地方忘了开`long long`导致WA。
    - **整改**：统一使用`ios::sync_with_stdio(false); cin.tie(0);`，并坚持用`cin/cout`。严格按需使用`long long`，除非题目明确涉及取模且值域可能溢出，否则禁用`#define int long long`。

2.  **数组大小随意与越界访问**
    - **症状**：数组开得“差不多”，通常是`maxn = 1e5 + 5`。在链表、建图时漏算了无向图的2倍边；在线段树中，数组只开了`4*n`但实际某些写法可能需要更多。
    - **整改**：养成习惯，在代码开头用注释精确计算所需最大空间：`// maxn = 1e5, maxm = 2e5 (undirected graph)`。对于数据结构，严格按公式 `4 * N` 并适当上浮。

3.  **STL容器重度依赖且不了解底层代价**
    - **症状**：在任何需要映射的场景都习惯用`unordered_map`，但却不知道其可能退化为\(O(n)\)，是导致TLE的隐形杀手。在需要频繁在尾部以外插入/删除时，仍使用`vector`。
    - **整改**：系统学习STL各容器的底层结构、复杂度及适用场景。能用数组解决的问题，优先用数组。使用`map`（红黑树，稳定\(O(log n)\)）替代可能退化的哈希表，除非你确信哈希函数足够随机。

🟢 **两个值得肯定的优点**

---

**教练示范题: [P1048 [NOIP2005 普及组] 采药](https://www.luogu.com.cn/problem/P1048)**

**a) AI 题解摘要**：
<p class="text-blue-700 font-semibold">这是一道经典的01背包问题。核心坑点在于：如果用DFS暴力枚举采或不采，会因\(O(2^n)\)的复杂度而指数爆炸。解决方法是发现“剩余时间”和“当前草药”的最优值是唯一的，从而引入DP进行记忆化。</p>

**b) 暴力思路怎么想？**
拿到题，最直接的“人脑模拟”法：从第一株草药开始，要么采，要么不采。采了，时间减少，价值增加；不采，时间和价值都不变。我们按此思路写出一个DFS函数 `dfs(i, t)` 表示处理到第`i`株，还剩`t`时间时的最大价值。
- **复杂度**：\(O(2^n)\)。n最多100，这简直是天文数字。
- **部分分**：大约能过 n <= 20 的数据点，预计得分 30分。

**c) 瓶颈在哪里？**
代码最大的瓶颈在于**大量的重复计算**。比如，我们通过路径 `采1 -> 不采2 -> 采3` 达到“处理完前3株，剩余时间是t”的状态，和通过 `不采1 -> 采2 -> 不采3` 达到“处理完前3株，剩余时间是t”的状态，它们在之后能获得的最大价值应该是**完全一样的**！但DFS没有记忆，它会傻傻地重新计算一遍。

**d) 关键性质/不变量观察 (Key Observation)**
仔细分析DFS，它有两个参数`(i, t)`。这两个参数，**唯一决定了后续所有操作能获得的最大价值**。我们称这个性质为 **“无后效性”**。一旦你确定了`i`和`t`，之后的最优解是固定的，与你是怎么走到`(i, t)`这个状态的无关。

**e) 最终正解的推导与核心代码结构**
既然`(i, t)`与最优解一一对应，我们可以建立一个二维数组 `dp[i][t]`，专门存储“处理到第`i`株，剩余`t`时间”这个状态下的最大价值。这就是**记忆化搜索**，再进一步，就是**动态规划**。

- **状态定义**: `dp[i][t]` 表示前 `i` 株草药，在总时间不超过 `t` 的情况下，能采到的最大总价值。
- **转移方程**:
  对于第 `i` 株草药，需要时间 `cost[i]`，价值 `val[i]`。
  - 如果不采：`dp[i][t] = dp[i-1][t]`
  - 如果采（前提是 `t >= cost[i]`）：`dp[i][t] = max(dp[i][t], dp[i-1][t - cost[i]] + val[i])`
- **核心代码结构**:
```cpp
for (int i = 1; i <= m; ++i) {
    for (int t = 1; t <= T; ++t) {
        dp[i][t] = dp[i-1][t];  // 不采
        if (t >= cost[i]) {
            dp[i][t] = max(dp[i][t], dp[i-1][t - cost[i]] + val[i]); // 采
        }
    }
}
```
- **复杂度**: \(O(n \times T)\)，完美解决。我们还可以用**滚动数组**将二维优化到一维。
