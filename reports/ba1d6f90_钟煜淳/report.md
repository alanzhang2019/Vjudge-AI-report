### Problem P1582 - 倒水

- **AI 题解摘要**: 看懂题意后，问题转化为：求一个最小的 `k`，使得 `N` 加上 `k` 后，其二进制表示中 `1` 的个数不超过 `K`。
- **暴力思路**: 从 `N` 开始，枚举 `k = 0, 1, 2, ...`，对于每一个 `N+k`，检查它能否被合并成不超过 `K` 个瓶子。
  - **如何检查？** 不断将两个相同水量 `x` 的瓶子合并成一个 `2x` 的瓶子。这个过程其实就是计算 `N+k` 的二进制表示中有多少个 `1`。例如 `13 (1101)` 有3个1，代表它最终可以变成 `8+4+1` 三个瓶子。
  - **复杂度**: `O(k * log(N+k))`。`k` 最大可能到 `2^31`，显然会超时。
- **瓶颈在哪里？**
  - 我们实际上是在一维空间里暴力搜索 `k`，效率太低。我们需要直接算出最小的合法 `N+k`。
- **关键性质/不变量观察**:
  - 合并过程等价于二进制加法。
  - 一个数可以被合并成的瓶子数量，等于它的二进制表示中 `1` 的个数（记为 `popcount(N+k)`）。
  - 我们要找的就是 `popcount(N+k) ≤ K` 的最小的 `k`（即大于等于N的最小数）。
- **最终正解推导**:
  - 问题转化为：从 `N` 开始，不断加上一个数，直到它的 `popcount` 不超过 `K`。最直接的方法不再是枚举所有数，而是利用位运算来“修补”二进制。
  - **一种更易懂的正解 (贪心)**:
    1.  从 `N` 开始。
    2.  计算当前数 `cur = N + k` 的 `popcount`。
    3.  如果 `popcount(cur) <= K`，则 `k` 即为答案，结束。
    4.  否则，我们需要减少 `popcount`。通过增加`k`，使得 `cur` 中最低位的 `1` 进位。例如 `cur = ...1001000`，我们加上 `...0001000` 使其变成 `...1010000`。这相当于把一堆低位的 `1` 合并成一个高位的 `1`。代码上，就是找到 `cur` 的最低位的 `1` 的位置 `lowbit = cur & -cur`，然后令 `k += lowbit`，并继续检查新的 `cur`。
  - **核心代码结构**:
    ```cpp
    int n, k, ans = 0;
    cin >> n >> k;
    int cur = n;
    while (__builtin_popcount(cur) > k) { // 也可以用自写函数算
        int lowbit = cur & -cur;           // 获取最低位的1
        ans += lowbit;                    // 答案增加
        cur += lowbit;                    // 补上这个数，触发进位
    }
    cout << ans << endl;
    ```

### Problem P11961 - `[GESP202503 五级] 原根判断`

- **AI 题解摘要**: 经典的数论问题，考点是：快速幂 + 约数枚举 + 原根定义的理解。你的代码思路是对的，但索引有bug。
- **暴力思路**: 给定 `a` 和 `p`，暴力枚举 `i = 1` 到 `p-2`，看 `a^i mod p` 是否都为1？显然，这不对。
  - **原根定义**: `a` 是模 `p` 的原根，当且仅当 `a^(p-1) ≡ 1 (mod p)`，且对于 `p-1` 的任意真因数 `d`  (`1 ≤ d < p-1`)，有 `a^d ≢ 1 (mod p)`。
  - **暴力**: 获取 `p-1` 的所有因数 `d`（除了 `p-1` 本身），逐个用快速幂检查，如果有一个为1，则不是原根。复杂度：`O(divisors * log(p))`。
- **瓶颈在哪里？**
  - 你的代码中，`find_yin` 函数只加入了 `i` 和 `x/i`，但 `x` 可能是平方数，`sqrt(x)` 会被加入两次。另外，你需要枚举所有因数，不仅仅是质因数。
  - 更大的bug在 `fun` 函数中：`for(int i = 1; i <= yin.size(); i++)` 数组索引从1开始，而 `vector` 下标从0开始！这会导致 `yin[yin.size()]` 数组越界，引发错误。
- **关键性质/不变量观察**:
  - 不需要检查所有因数，只需要检查 `p-1` 的 **所有质因数** `q`。因为如果一个数 `d` 使得 `a^d ≡ 1`，那么 `d` 的倍数也会。因此只需检查最小的“模数”，即 `(p-1)/q`。
- **最终正解推导**:
  1. 预处理 `p-1` 的所有质因数。
  2. 检查 `a^(p-1) % p == 1`。不是则直接 false。
  3. 对于每个质因数 `q`，计算 `a^((p-1)/q) % p`，如果等于1，直接返回 false。
  4. 全都不为1，则是原根。
